## 1. 无痛取材

不要从「你想写什么」开始审问。先识别用户已经给出的最小素材：

| 用户手里有什么 | 立即做什么 |
|---|---|
| 一句话或情绪 | 找出它对应的具体场景和被卡住的人 |
| 语音或随手记 | 提取反复出现的问题、最有力度的原话和一个核心判断 |
| 一个专业概念 | 翻译成读者会经历的时刻、误解和微行动 |
| 一个常见提问 | 区分表面问题、真实担心和读者此刻能做的下一步 |
| 作者自己的经历 | 找出过去信念、转折、现在的工作判断和适合帮助的人 |
| 聚合工作观察 | 只写反复出现的模式，不写任何人的可识别故事 |
| 一项服务 | 写清适合谁、解决范围、过程、边界和低压力入口 |
| 什么都没有 | 不问抽象定位。只问「最近一次有人找你聊事，他当时原话怎么问的？」；若也没有，先从作者自己的一个真实经历开始 |

素材杂乱时先做「一核三枝」：

- **一核**：这次只讲哪一个判断；
- **枝 1**：读者在哪个时刻会需要它；
- **枝 2**：作者凭什么做出这个判断；
- **枝 3**：读者看完能完成哪个最小动作。

默认先做一条，不让用户先填长表。批量任务再把同一核心拆成系列。

如果作者完全卡住，每次只抛一个可用问题，并接着完成内容：

- 「这周哪一个问题被反复问到？」
- 「哪一种状态最让你想提醒，但又不想说教？」
- 「你以前相信、现在不再完全相信的一个判断是什么？」
- 「大家最容易误解你的服务哪一点？」
- 「如果只允许读者今天做一个小动作，你会选什么？」

不要连续抛出五问。用户回答一句后，Skill 承担提炼和成稿工作。

## 2. 咨询师个人 IP 与记忆点定位

定位一律使用 `positioning-brain.md`。本文件只负责把已经有现实依据的方向变成选题。

禁止从职业、性格、兴趣、证书或作者想象的人群直接生成定位。用户说“尖锐”“大实话”“很多人聊完会哦～”时，只把它当线索：可能擅长重新定义问题。继续找一段真实对话，确认对方原来怎么想、作者指出了什么、后来做了什么。

没有真实求助和服务时，不输出“你的定位是”。写成“目前最值得测试的是”，并用内容和真实对话验证。

## 3. 内容资产地图

咨询师、家庭教育指导师和其他助人专业者的稳定内容不应只来自个案。优先从以下八类安全资产取材：

1. **被看见**：说出目标读者正在经历却难以描述的时刻；
2. **机制理解**：解释一个反应可能怎样形成或被维持；
3. **区别判断**：区分两个容易混淆的状态、需求或选择；
4. **微小练习**：给一个低风险、可停止、不过度承诺的观察或行动；
5. **引导式提问**：用问题帮助读者看见目标、现实、选项与下一步；
6. **作者来时路**：写自己的经历、变化和专业选择，不借来访者故事证明自己；自我披露要服务读者，不把尚未处理的情绪交给读者承接；
7. **服务过程与边界**：解释什么时候适合咨询或指导服务、会怎样开始、什么情况应转介；
8. **专业祛魅**：纠正常见误解，诚实说明方法的价值、限制和个体差异。
9. **产品教育**：说明一项服务处理什么、不处理什么、双方如何合作以及适配条件；
10. **购买顾虑**：诚实回答隐私、费用、时机、效果预期和依赖等真实担心。

形成长期方向时，选择 3 至 5 类即可。对用户不要说“内容支柱”；直接说“以后主要反复讲这几类事”。至少保留一类服务过程或产品说明，让专业可信度不只靠知识科普。

## 4. 读者处境六格

写作前把抽象人群落到一个时刻：

1. **可见场景**：他在哪里，刚发生了什么；
2. **脑内原话**：他会怎样对自己说；
3. **情绪与身体**：他可能感觉到什么；
4. **保护动作**：逃避、讨好、控制、反复确认等行为在试图保护什么；
5. **需要或价值**：安全、连接、自主、胜任、尊重或意义；
6. **最小移动**：今天能多看见什么、问什么或做什么。

六格是创作假设，不是对个人的远程评估。正文使用「可能」「有时」「对一些人来说」，并给读者保留不同解释。

## 5. 心理学观察镜头

每条内容只选一个镜头。镜头用于组织观察，不用于下诊断。

### 保护功能

适合写「明知没用却停不下来」的行为。结构：行为 → 它曾试图保护什么 → 当前代价 → 更温和的新选择。

### 情境循环

适合写反复卡点。结构：触发情境 → 自动解释 → 情绪/身体 → 反应 → 短期缓解 → 长期后果。不要把一种循环写成所有人的原因。

### 需要、依恋与边界

适合关系议题。先写具体互动，再提出对安全、连接、自主或边界的可能需要。避免凭一条行为给读者或其伴侣贴依恋、人格或障碍标签。

### 矛盾心理

适合「一边想变，一边动不了」。同时尊重想改变的一部分和维持现状的一部分，寻找两者各自保护的价值。

### 价值—行动落差

适合个人成长和生涯议题。区分「我真正重视什么」与「我为了避免不适正在做什么」，再设计一个很小的价值行动。

### 自我慈悲

适合羞耻和自责。先承认难度与共同人性，再区分理解自己和免除责任，最后回到可以承担的行动。

## 6. 咨询式提问与行动镜头

行动类内容以恢复选择权和行动感为主，不把公开内容包装成个体咨询或治疗。

### 目标—现实—选项—承诺

- 你真正想要的变化是什么？
- 现在客观发生了什么？
- 除了惯性做法，还有哪些选项？
- 下一步愿意在何时做什么？

适合职业、目标、习惯和决策内容。若读者出现明显临床或危机风险，不用行动框架覆盖其求助需要。

### 量尺问题

用 0—10 描述当前位置，追问「为什么不是更低」「上升 1 分会出现什么」。重点是找已有资源和可观察变化，不制造伪精确测量。

### 理想未来

让读者描述问题缓和后的一个普通早晨、一次对话或一个工作日，再倒推第一个可见信号。避免宏大空想。

### 可控圈与小实验

区分不可控、可影响、可直接行动三层，把建议改成有期限、可观察、允许失败的小实验。

### 资源回看

回看读者曾应对类似处境的时刻：当时做了什么、谁支持过、哪项能力仍可调用。不要用「你其实什么都能做到」否认现实约束。

## 7. 作者判断提纯

内容不够聪明，常常不是信息太少，而是没有形成作者自己的判断。以下四问只在内部使用：

1. 大家通常怎样解释这个问题？
2. 作者根据什么不同意、补充或重新定义？
3. 旧解释会让人做出什么无效动作或付出什么代价？
4. 新判断会改变读者的哪个下一步？

优先保留用户原话中有力度的词。把锋芒对准误区、惯性动作或行业问题，不对准脆弱的人。强判断必须能给出场景、推理、经验或来源，并允许例外。

成稿前写出一句内部判断，不展示给用户：

```text
这篇不是在讲【泛知识】，而是在提醒【具体人】：真正先要处理的是【关键问题】，因此下一步应从【动作】开始。
```

## 8. 专业话转人话

按以下顺序翻译：

```text
术语或理论
→ 读者会遇到的具体场景
→ 他脑中会出现的一句话
→ 这个反应可能在保护什么
→ 继续下去的现实代价
→ 一个可尝试的小动作
→ 方法的适用边界
```

例：不要只写「建立边界」。写「同事临时把工作推给你，你嘴上答应，回家后越想越委屈。边界不是马上拒绝所有人，而是先给自己一句缓冲：『我看一下今天的安排，半小时后回复你。』」

翻译检查：

- 删除读者不需要的流派史和术语堆叠；
- 保留关键区别，不把复杂问题压成万能口诀；
- 把「你是」改成「你可能正在」；
- 把「应该」改成可选择的观察、提问或实验；
- 把「有效」改成对谁、在何种条件下、可能有什么帮助。

## 9. 一份素材多次复用

同一核心判断可生成五种资产，但不要机械复制：

1. **小红书发现稿**：从具体处境和识别感进入；
2. **短视频判断稿**：一句反常识 + 一个场景 + 一个行动；
3. **朋友圈信任稿**：作者观察 + 工作价值观 + 轻邀请；
4. **公众号深度稿**：问题、机制、边界、例子和完整路径；
5. **服务说明稿**：适合谁、不适合谁、过程、双方责任和下一步。

改编时保留同一个核心判断，重写开头、证据密度、篇幅和 CTA。默认先完成用户当前平台，再建议一个最值得复用的方向。

## 10. 七次无痛循环

用户想稳定输出时，用一个核心主题排七次表达，不强制日更：

1. **被看见**：写一个目标读者正在经历的时刻；
2. **区别判断**：区分两个常被混淆的状态；
3. **机制或引导问题**：根据主题解释一个可能循环，或展开目标、现实与选项；
4. **作者来时路**：只写自己的经验与判断变化；
5. **微小练习**：给一个低风险、可停止的小动作；
6. **服务过程与边界**：解释怎样开始、适合谁、何时转介；
7. **问题回收**：回答公开区高频问题，形成下一轮选题。

一次只生产当前最容易的一条。用户要求一周计划时再输出完整循环，并让七条共享同一个核心主题而非七个散点。
