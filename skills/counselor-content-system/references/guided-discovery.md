# 从空白到第一条内容

用户说“不知道写什么”“脑子里一片空白”“我没素材”时，完整读取本文件。目标不是让用户完成自我分析，而是用最少提问拿到一个真实素材，并在当前对话里做出第一条内容。

## 1. 零资料也能开始

不要要求用户先建知识库、整理案例、列内容方向、做用户画像或填写问卷。先使用当前对话里已经出现的职业、服务、原话和情绪。

若仍没有素材，按下面顺序只选一个入口：

1. **最近的真实提问**：这周有人找你聊过什么？他原话怎么说？
2. **最近的不认同**：最近看到哪句话，你很想说“事情不是这么简单”？
3. **反复出现的误解**：别人最容易误会你的服务哪一点？
4. **自己的真实经历**：最近哪件事让你改了一个看法？
5. **当前服务**：如果只让一个人知道你能帮什么，你最想先说哪件事？

不要把五个入口一次发给用户。根据已有信息挑最容易回答的一个。

## 2. 一次只问一个小问题

坏问题：

```text
你的目标用户是谁、核心痛点是什么、你有什么差异化方法、想在哪个平台变现？
```

好问题：

```text
最近一次有人找你聊完，真的决定去做的那一步是什么？
```

一个好问题应满足：用户能从记忆里找答案；答案会改变选题或判断；没有营销术语；一句话就能答。

## 3. “不知道”就继续降低难度

连续追问最多三轮，每轮只降一级：

1. “你通常帮助谁？”降成“最近谁来找过你？”
2. “他遇到什么问题？”降成“他开口的第一句话是什么？”
3. “你用了什么方法？”降成“你第一反应让他先别做什么，或者先做什么？”

如果用户仍然想不起真实对话，不要继续盘问。根据其职业和一个常见低风险场景提出临时选题，明确说这是测试稿，不是假装真实案例，然后直接成稿。

## 4. 拿到一句话后怎么继续

内部提取五件事，不把表格展示给用户：

```text
发生了什么
→ 对方原来以为什么
→ 作者指出了什么
→ 下一步发生了什么变化
→ 这件事和作者的服务有什么关系
```

缺最后一项也可以先写判断型内容；不要为了塞销售而毁掉素材。

## 5. 什么时候不再提问

符合任一情况就直接做：

- 已有一个具体场景和一句作者判断；
- 用户明确要“一条能发的内容”；
- 继续提问只会让用户定义抽象概念；
- 已经问到第三轮；
- 用户明显不耐烦或说“你先写一个我看看”。

交付后如确实需要校准，只补一个轻问题，例如：“这版里哪一句最像你平时会说的话？”不要再发一套素材采集表。

## 6. 系统主动承担的工作

用户只负责提供碎片，系统负责：

- 从碎片中提炼选题，不让用户自己命名赛道；
- 区分事实、作者判断和系统假设；
- 把抽象专业词换成场景、动作和原话；
- 选择当前最适合的平台写法；
- 在不虚构、不越界的前提下补足结构；
- 信息不足时给可验证的小样，而不是假装确定。

“无痛输出”不是不需要真实材料，而是不把整理、归纳和翻译的劳动退回给用户。

## 7. 首次使用参考回复

完全空白时：

```text
不用先想定位，也不用先建素材库。你说得乱也可以，我来整理。

我们先从一件真的发生过的事开始：最近一次有人找你聊事，他开口的第一句话是什么？
```

用户连这也想不起时：

```text
那先不找客户故事。最近有没有一句同行常说的话，让你很想反驳？随便说一句，我把它做成第一条。
```

第三轮仍空白时：

```text
先不继续问了。我按你现在的职业做一条测试稿。它不是你的正式定位，也不冒充真实案例；我们先看你对哪一句有感觉，再往回找你的材料。
```
