# 咨询师与教练商业化内容引擎

在定位、服务介绍、产品教育、内容获客、信任建立或引导咨询任务中完整读取本文件，同时读取 `commercial-first-principles.md` 与 `plain-language-and-voice.md`。本文件的分类只供内部判断，不要把“商业断点、付费场景、内容任务、信任支撑、低压力入口”等词说给用户。商业化目标服从 `ethics-and-safety.md`：不靠诊断、恐惧、依赖、虚构证言或结果保证促成购买。

## 1. 先诊断商业断点

不要默认用户缺文案。先判断当前断点：

| 表面问题 | 常见真实断点 | 优先交付 |
|---|---|---|
| 不知道写什么 | 缺少可用的真实素材，或不会从日常工作里识别选题 | 按 `guided-discovery.md` 问出一个素材 + 第一条测试稿 |
| 内容很专业但没人咨询 | 只证明懂知识，没说明与谁有关、能如何帮助 | 用户场景化改写 + 价值翻译 |
| 不知道怎么卖自己 | 把销售理解成自夸，没把能力变成可判断的路径 | 服务价值链 + 适配说明 |
| 有阅读收藏但没线索 | 内容没有下一步，主页或入口断裂 | 线索型内容 + CTA + 入口建议 |
| 一发广告就别扭 | 信任内容不足，产品表达只剩价格与名额 | 产品教育稿 + 过程透明 |
| 很多人问但很少成交 | 用户问题模糊、服务边界或适配关系不清 | 诊断入口 + 服务分流 |
| 想做个人 IP 但人群太宽 | 用职业或性格代替了记忆点 | 记忆点定位 + 小样本验证 |

信息不足但风险不高时，给出“目前我更倾向于…… + 一个能直接使用的版本 + 一个事实问题”，不要展示内部诊断词，也不要把工作退回给用户。

## 2. 记忆点定位

定位以 `positioning-brain.md` 为准。本文件只检查一个已有现实依据的方向能不能变成客户听得懂的内容和服务。

内部检查：

- 现实里是否真的有人带着这类问题来过；
- 作者是不是反复做对了同一种判断或推进；
- 作者现在是否有服务能接住；
- 读者能否立刻说出“我在什么情况下会找他”。

“尖锐”“温柔”“真诚”只描述说话方式。没有真实求助、真实服务和真实反馈时，不替用户宣布定位，只给一条测试方向。

## 3. 专业价值翻译

用户不会购买抽象的“专业能力”，也不是在购买作者本人。他会判断这项帮助是否与自己的处境有关、过程是否可信、投入是否值得。

按以下链条翻译：

```text
我会什么
→ 用户在什么场景遇到什么问题
→ 我实际帮助处理哪个工作对象
→ 我会怎样判断与推进
→ 用户可能获得什么可观察变化
→ 需要用户投入什么
→ 有哪些限制与不适配情况
```

示例：

```text
弱：我提供生涯教练服务。
强：如果你不是没有选项，而是每个选项都担心选错，我会先帮你拆开“真正想要、现实约束和害怕承担的代价”，再把决定缩小成一个可验证的小实验。它适合需要梳理和行动的人，不替你做决定，也不保证某个职业结果。
```

“卖结果”要改写为“让潜在价值和路径可见”。可以描述希望推进的方向和可交付过程，不把复杂的人生或心理变化包装成确定疗效。

## 4. 七种商业内容任务

每条内容只选一个主任务，最多带一个辅任务。

| 任务 | 读者看完发生什么 | 内容重点 | 合适下一步 |
|---|---|---|---|
| 被发现 | 在具体问题中停下来 | 场景、冲突、搜索词 | 看下一篇 / 收藏 |
| 被认出 | 产生“这说的是我” | 脑内原话、情绪与现实代价 | 自我观察 |
| 建立判断 | 发现旧解释不够 | 区别、反常识、推理链 | 做一个小检查 |
| 证明可信 | 看见作者不是只会说结论 | 依据、方法、边界、过程 | 查看主页或相关内容 |
| 服务教育 | 知道什么时候适合找何种帮助 | 服务对象、过程、责任、限制 | 了解 FAQ / 适配说明 |
| 处理顾虑 | 一个真实购买阻力被说清 | 费用、时机、隐私、是否适合、预期 | 做适配判断 |
| 引导行动 | 愿意迈出低压力下一步 | 入口价值、会发生什么、隐私提示 | 私信关键词 / 填表 / 预约 |

不要每条都直接预约。若信任尚浅，先让读者确认“你懂我的处境”和“你的判断对我有用”。

## 5. 信任支撑系统

助人服务的信任不能只靠证书，也不能依赖来访者故事。按主张选择支撑：

### 五类可信资产

1. **判断资产**：对一个常见误区给出清楚、有推理的区别；
2. **过程资产**：展示如何开始、如何评估适配、怎样推进、何时暂停或转介；
3. **专业资产**：真实训练、资质、工作范围、持续学习与公开专业规范；
4. **人格资产**：作者自己的来时路、工作价值观、稳定边界与真实生活状态；
5. **交付资产**：不泄密的工具、框架、会谈准备、阶段回顾、可验证交付物与服务说明。

### 证据强度匹配

- 说个人判断：给推理和反例；
- 说反复模式：说明来自聚合观察，不代表所有人；
- 说方法有帮助：说明对谁、在何种条件下、帮助什么、有什么限制；
- 说研究结论：优先原始研究、系统综述或权威机构，不引用不明二手金句；
- 说服务有效：避免单一证言和前后对比代替证据，优先展示过程、适配与可交付物。

若一篇内容只有态度没有支撑，补一个场景、推理、过程、公开依据或作者真实经历。若支撑会暴露来访者，换支撑，不做“高级脱敏”冒险。

## 6. 从内容到客户的最短路径

商业内容不是单篇成交，而是让读者完成连续判断：

```text
内容：这和我有关
→ 主页/连续内容：这个人专门理解这类问题
→ 信任支撑：他的判断和工作方式可信
→ 低压力入口：我可以先了解或自查
→ 适配判断：这项服务适合我现在的需要
→ 正式服务：过程、边界、责任和费用清楚
```

写稿前找出当前链路断点，只修一个：

- 内容看不懂：改为具体场景；
- 看懂但记不住作者：强化稳定记忆点和独特判断；
- 信任不足：补过程、依据、边界或作者来时路；
- 有兴趣不行动：设计与正文一致的低压力入口；
- 线索多但不适配：提前讲清适合、不适合与转介；
- 咨询多但成交弱：把模糊问题诊断具体，把产品路径讲清。

## 7. 服务与产品表达

服务不是简历，也不只是“一小时多少钱”。用八要素表达：

1. 谁在什么时刻适合考虑；
2. 当前要处理的具体问题；
3. 服务目标与不可承诺部分；
4. 大致步骤、频率、形式或周期；
5. 从业者提供什么判断、支持或反馈；
6. 客户需要投入什么行动与信息；
7. 适合、不适合、暂停与转介条件；
8. 如何低压力了解费用、协议和预约流程。

### 入口设计

按准备度选择，不制造虚假诊断：

- 一份非诊断型自查清单；
- 服务区别与适配 FAQ；
- 公开文章或短课；
- 不收集敏感病史的初步问卷；
- 体验会谈、化学会谈或正式预约流程。

入口必须说明“用户会得到什么、不会得到什么、是否收集资料、下一步是什么”。不要用免费测评伪装诊断或强行销售。

## 8. 尖锐但不伤人的观点提炼

尖锐内容的对象应是误区、惯性解释、行业问题或无效动作，不是脆弱的读者。

用四步形成“大实话”：

1. 读者/同行常用什么合理说法解释现状；
2. 这个说法遮住了什么更难面对的事实；
3. 继续相信它会付出什么代价；
4. 更诚实、可执行的替代判断是什么。

结构：

```text
很多人说【表面解释】。
但真正卡住你的，可能不是【常见归因】，而是【被遮住的选择/代价】。
如果一直不说清楚，【现实后果】。
更有用的下一步是【具体动作】。
```

避免机械使用“不是……而是……”。保留作者原话、节奏和真实力度。每个强判断都要能被解释，允许例外，不把羞耻当钩子。

## 9. 商业选题生成

围绕一个服务生成内容时，从以下问题各取一条，不要只写知识科普：

1. **付费时刻**：什么发生后，读者开始认真找帮助？
2. **错误归因**：他以为问题在哪，作者判断真正应先看什么？
3. **判断标准**：如何区分两个容易混淆的状态或服务？
4. **过程透明**：第一次接触、一次服务或一个阶段通常怎样进行？
5. **适配边界**：谁适合、谁暂时不适合、何时需要转介？
6. **行动障碍**：他为什么知道却动不了，下一步怎样缩小？
7. **服务误解**：客户最容易误解服务的哪一点？
8. **购买顾虑**：对隐私、费用、效果、依赖或时机的担心如何诚实回答？
9. **作者判断**：作者最不同意行业里哪句看似正确的话？
10. **工具入口**：什么非诊断型清单、问题或小实验能让读者体验作者判断力？

筛选时优先：场景具体、代价真实、作者能承接、能展示判断、下一步自然。热点只在与目标人群和服务路径相关时使用。

## 10. 发布与校准

发布前记录：目标读者、主任务、最强判断、使用的支撑、希望发生的下一步、最大风险。

发布后区分四类信号：

- **停留信号**：完读、收藏、转发、重复提及某句话；
- **精准信号**：目标人群的评论、主页访问、相关私信；
- **信任信号**：询问方法、边界、经历、服务过程；
- **商业信号**：领取入口、完成自查、询问适配、预约或购买。

一条数据不定义定位。连续观察 10—20 条：哪类场景吸引对的人、哪种判断被记住、哪种入口产生高质量行动，再更新内容支柱与服务表达。没有条件记录时，只交付轻量复盘问题，不强迫建库。

每次给出商业化产物后，按 `action-loop-and-handoff.md` 选择一个现实动作和一个观察点。不要只说“建议发布测试”，要说清发什么、给谁、看什么。问题进入连续定位、产品、销售与行动调整时，先完成本轮小结果，再说明真人咨询或陪跑的边界。
