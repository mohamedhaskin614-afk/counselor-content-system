# 商业化第一原则

涉及定位、内容获客、服务介绍、产品、咨询转化或“为什么没人来问”时，先完整读取本文件，再按需读取 `commercialization-engine.md`。这些原则只用于内部判断，输出必须说普通话。

## 1. 内容不是能力展览

内容的首要作用不是证明作者懂很多，而是帮助一个具体的人完成判断：

```text
这是不是我的问题
→ 我原来哪里理解错了
→ 我现在先做什么
→ 这个人怎样帮
→ 我是否需要进一步了解
```

如果一篇内容只能让同行看出作者学过什么，却不能让客户认出自己、看懂帮助或决定下一步，它的商业作用很弱。

## 2. 专业不是复杂，专业是区分得准

真正的专业可以表现为：

- 把两个容易混淆的问题分开；
- 指出表面问题后面真正要先处理的事；
- 讲清判断依据、工作过程和边界；
- 知道什么能帮、什么不能保证、何时应转介；
- 把一个大问题压到当下能做的下一步。

术语、流派史和复杂句子只有在会改变读者理解或行动时才保留。一条内容默认只承担一个专业概念。

## 3. 复杂化常常是在回避真问题

遇到堆术语、反复定义、长篇铺垫或“高级表达”时，先问内部问题：

```text
如果把术语拿掉，作者到底想提醒读者什么？
```

再检查：

```text
这句话说给一个正在经历这件事的人，他会知道这和自己有什么关系、下一步做什么吗？
```

若不会，说明不是读者不专业，而是作者还没有把痛点拆明白。不要润色复杂话，先重新判断问题。

## 4. 客户购买的是一段清楚的帮助过程

客户不是购买作者本人、知识数量、证书墙或一小时聊天。他在判断：

- 我现在的事情是不是你处理的范围；
- 你会先看什么、怎样推进；
- 我需要投入什么；
- 可能得到哪些可观察的进展；
- 哪些结果你不能替我决定或保证；
- 费用、时间和风险是否值得。

心理咨询、教练与家庭教育服务不能把复杂变化承诺成确定结果。可以卖“清楚的过程、专业判断、阶段产物和合作方式”，不能卖“保证治好、保证改变或保证成功”。

## 5. 销售是帮助对方做决定

销售不是逼人买，也不是藏着价格等对方悟。它至少要让对方看清：

```text
适合谁
→ 解决范围
→ 怎样开始
→ 双方各自做什么
→ 不适合谁
→ 下一步如何了解
```

好的内容既吸引合适的人，也让不合适的人提前退出。筛选不是损失客户，是保护服务关系。

## 6. 没有服务时不要假装转化

若用户还没有真实服务、交付经验或基本边界：

- 不硬写“欢迎预约”；
- 不凭空包装高价产品；
- 不宣布已经验证的人群定位；
- 先用内容测试问题是否真实、作者是否能持续给出有用判断；
- 先做低风险访谈、公开答疑、小型工作坊或明确边界的测试服务。

商业化不是尽快收钱，而是让价值、交换和责任都变清楚。

## 7. 一条商业内容的最短骨架

内部按需使用，不要机械套成五段：

```text
用户刚发生了什么
→ 他通常会怎样理解
→ 作者认为真正先要处理什么
→ 作者会怎样帮助推进
→ 读者下一步如何判断
```

不同内容可以只展开其中两三步。不要每条都加销售结尾；当正文已经说明服务过程或适配条件时，邀请才自然。

## 8. 交付前商业检查

逐条检查：

1. 目标读者能否在前几句认出自己的处境？
2. 作者的判断是否具体，而不是只展示态度或术语？
3. 读者是否知道这项帮助与自己有什么关系？
4. 有没有把服务说成作者有多厉害，而不是客户会经历什么？
5. 下一步是否符合当前信任程度？
6. 有没有制造恐惧、依赖、羞耻或虚假确定性？

只修最先断掉的一处，不为了“完整闭环”把一条内容写成说明书。
