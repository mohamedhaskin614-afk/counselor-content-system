# 说人话与本人感

写稿、改稿或输出诊断前完整读取本文件。目标是让读者第一遍就懂，不需要作者再解释“我的意思是”。

## 1. 框架只给系统看

以下词可以用于内部思考，默认不要出现在交付给用户的文案里：

| 内部词 | 说给人听 |
|---|---|
| 商业断点 | 具体卡在哪一步 |
| 付费场景 | 他遇到什么事会开始找人帮忙 |
| 价值翻译 | 把你会的说成客户听得懂的帮助 |
| 可信支撑 | 你凭什么这么说 |
| 内容任务 | 这条发出来要起什么作用 |
| 认知移动 | 读者看完明白了什么 |
| 适配关系 | 谁适合，谁不适合 |
| 低压力入口 | 他下一步可以怎么找你 |
| 内容支柱 | 以后主要反复讲哪几类事情 |
| 用户画像 | 具体是谁，他最近遇到了什么 |
| 信任资产 | 什么内容会让别人慢慢相信你 |

“承接、闭环、赋能、链接、抓手、颗粒度、路径、底层逻辑”也要慎用。能用动作说清楚就不用名词。

## 2. 抽象词落地

抽象词出现后，必须马上回答“具体是什么样”。

```text
差：帮助家长提升亲子沟通能力。
好：孩子一关门，家长先忍住追着问，等情绪下来后再说：“你现在想自己待会儿，还是愿意告诉我发生了什么？”
```

```text
差：帮助来访者建立边界。
好：同事临时把活推过来时，不再下意识答应。先回一句：“我看一下今天的安排，下午三点前答复你。”
```

```text
差：找回行动力。
好：不再继续做计划，今天先把报名页面发给第一个人看。
```

## 3. 识别“用复杂证明专业”

出现以下信号时，不要只做遣词润色：

- 一句话塞进三个以上专业名词；
- 先讲流派、模型和机制，迟迟不说读者发生了什么；
- 用“系统性、深层次、全方位、底层”等词代替具体判断；
- 作者担心说简单了显得不专业；
- 读者看完只能说“好厉害”，却不知道这和自己有什么关系。

先在内部回答：

```text
一个正在经历这件事的人，第一遍能否知道：你在说我、你提醒了什么、我下一步怎么办？
```

若不能，先拆问题，不给复杂句子抛光。只保留会改变理解或行动的一个专业概念，其余改成场景、区别、动作、过程或边界。

专业感不靠术语密度。它来自一句话把问题分得比别人准。

## 4. 不替读者演戏

不要凭空写：

- “你是不是也经常在深夜辗转反侧？”
- “你内心深处其实很渴望被爱。”
- “看到这里的你一定已经泪流满面。”

可以写看得见的事：

- “消息发出去十分钟没回，你已经打开对话框看了五次。”
- “明明不想答应，嘴上还是先说了好。”
- “计划表改了三版，真正要发的那条内容还没动。”

看得见的事让读者自己认领，不需要作者替他下结论。

## 5. 保留人的毛边

优先保留用户原话中的：

- 突然冒出来的比喻；
- 吐槽和不耐烦；
- 半句自我纠正；
- 真正犹豫的地方；
- 具体人名以外的生活细节；
- 说得不漂亮但很准的一句话。

不要把“像哈达一样的待办清单”“勤奋货架”“哦～这样”改成“系统化交付”“产品矩阵过载”“认知发生转变”。后者更整齐，也更没味道。

## 6. 控制 AI 味

重点删：

- 连续三次以上“不是……而是……”；
- 每段都用金句收尾；
- 三点、三层、三步反复出现；
- “在这个快节奏的时代”“真正的……从来不是……”；
- 没有来源的“很多人都”“研究表明”；
- 虚构朋友、虚构学员、虚构来访者；
- 结尾突然升到人生、时代或命运；
- “你值得被爱”“愿你成为更好的自己”。

不要为了去 AI 味故意写错字或装口语。先把事情想清楚，再用作者会说的话写。

## 7. 不同交付怎么说

### 用户要一条朋友圈

直接给完整朋友圈。除非有安全问题，不加创作说明、结构拆解和三组标题。

### 用户要定位

像对话，不像报告。先说当前判断，再问一个事实问题。不要给四宫格和十条内容支柱。

### 用户要诊断文案

直接引用最有问题的一句，说明为什么读者会看不懂或划走，再给改法。不要只说“缺乏场景感”。

### 用户要服务介绍

写清谁会来、通常卡在哪、服务里会做什么、用户要做什么、什么情况不适合、如何开始。不要把它写成资历墙。

## 8. 交付前朗读检查

逐句检查：

1. 目标读者不懂营销和心理学，第一遍能懂吗？
2. 这句话嘴上会有人这样说吗？
3. 把抽象名词删掉，意思还在吗？
4. 有没有一句需要作者补充解释？有就重写。
5. 有没有在替读者下诊断或替作者编经历？
6. 最重要的一句话是哪句？其他句子有没有抢戏？
7. 读者看见的是作者懂很多，还是作者真的懂他现在发生了什么？

不通过就继续改，不把半成品交给用户解读。
