## 1. 选题判断

先写清：

- **读者时刻**：谁在什么具体情境下需要这条内容；
- **真实矛盾**：他既想要什么，又在保护或害怕什么；
- **内容任务**：被理解、理解机制、做出判断、尝试行动或了解服务；
- **商业任务**：被发现、被认出、证明可信、服务教育、处理顾虑或引导行动；
- **作者依据**：作者经历、聚合观察、专业训练、公开研究或明确标注的合成情境；
- **安全边界**：不公开什么，不承诺什么，何时建议寻求个体化帮助；
- **一个核心判断**：全篇只完成一次认知移动。

发布判断分三类：

- **直接写**：有明确读者时刻、作者依据和安全表达；
- **先补证据**：涉及研究、数据、诊断或方法有效性，但没有可靠来源；
- **换素材**：依赖可识别个案、当前来访者反馈、危机事件或越界承诺。

不要因为某个话题「有流量」就忽略作者资质与受众脆弱性。

## 2. 八种成稿结构

根据内容需要选择一种，不把八套全塞进一篇，也不为了套结构破坏用户原话。涉及线索、产品或服务时，先读取 `commercialization-engine.md` 找出具体卡点。

### A. 被看见型

```text
具体时刻
→ 读者脑内原话
→ 温和命名其矛盾
→ 一个不贴标签的理解
→ 一个最小动作
→ 轻 CTA
```

适合焦虑、关系、内耗、拖延、迷茫等高共鸣话题。先接住，再解释；不要用共情语言假装读心。

### B. 心理机制型

```text
常见但具体的现象
→ 一个可能的维持循环
→ 这个反应的短期保护
→ 长期代价
→ 打断循环的一个入口
→ 个体差异与求助边界
```

适合心理教育。引用研究时说明来源和适用范围，不把相关关系写成因果。

### C. 引导提问型

```text
当前卡点
→ 想要的变化
→ 现实与约束
→ 2—3 个有区分度的问题
→ 一个可观察的小实验
→ 复盘提示
```

适合目标、职业、决策、习惯与行动。问题要扩展选择，不诱导读者接受作者预设答案。

### D. 区别判断型

```text
两个容易混淆的状态
→ 各自的可观察信号
→ 为什么会混淆
→ 分别适合怎样回应
→ 什么时候需要个体化评估
```

适合「休息还是逃避」「支持还是拯救」「家庭指导还是心理咨询」等话题。避免制作自诊断清单。

### E. 作者来时路型

```text
过去的一个真实场景
→ 当时相信什么
→ 什么经历让判断变化
→ 现在如何工作或选择
→ 这段经历让作者更懂哪类人
→ 不神化个人经验的边界
```

只能写作者自己的经历或已公开且获授权材料，不把来访者经历偷换成第一人称。

### F. 服务说明型

```text
什么人会在什么时刻考虑这项服务
→ 服务能帮助澄清或推进什么
→ 大致过程与双方责任
→ 适合 / 不适合
→ 不能保证什么、何时需要其他专业支持
→ 一个低压力下一步
```

适合咨询、家庭教育指导、工作坊和体验会谈介绍。卖的是清楚的过程与潜在价值，不是保证发生的结果。

### G. 线索诊断型

```text
目标读者正在经历的具体断点
→ 他通常怎样误判问题
→ 作者给出一个更准确的判断
→ 2—4 个可观察信号或检查项
→ 一个不替代专业评估的小动作
→ 与正文一致的低压力入口
```

适合“有收藏没咨询”“孩子一写作业就冲突”“想行动却一直准备”等问题。公开内容只帮助读者把问题具体化，不生成心理或医疗诊断。

### H. 顾虑回应型

```text
一个真实但不泄密的购买顾虑
→ 先承认顾虑合理在哪里
→ 说明常见误解
→ 展示服务过程、边界或适配标准
→ 谁适合 / 谁不适合
→ 自主决定的下一步
```

适合费用、隐私、效果预期、依赖、是否值得开始等问题。不要伪造“客户都这样问”，可写成公开 FAQ 或作者主动说明。

## 3. 平台适配

### 小红书图文

让封面、标题、首段和正文承诺同一件事。推荐推进：

```text
封面：具体处境或关键区别
第 1 屏：读者正在经历什么
第 2 屏：常见误解
第 3—5 屏：一个判断或机制
第 6 屏：一个自查/练习
末屏：边界 + 轻 CTA
```

自查用于自我观察，不给读者生成诊断结论。标签只选与人群、场景和问题真正相关的词。

### 短视频 / 口播

一句只承载一个意思，先说场景再说概念：

```text
3 秒：具体处境或反直觉判断
→ 为什么这和读者有关
→ 一个场景或机制
→ 一句可记住的区别
→ 一个动作或问题
```

读出成稿，删掉书面连接词、机械排比和无法一口说完的句子。不要用危机、创伤或诊断词做猎奇钩子。

### 朋友圈

朋友圈要像一个真实的人在说一件今天值得说的事，不写成小论文，也不写成硬广墙：

```text
今天观察到的一个现象
→ 我的判断或工作价值观
→ 它对读者意味着什么
→ 可选的轻邀请
```

内容轮换：专业判断、作者生活与来时路、服务过程与边界、产品教育、购买顾虑、公开知识、活动说明。对用户直接交付成稿，不附“这条属于观点型、信任型”等说明，除非用户要诊断。不要发布可被来访者认出的「刚刚咨询完」故事。

### 公众号 / 长文

围绕一个问题形成完整理解：

```text
读者处境
→ 为什么常见答案不够
→ 一个清楚机制或区别
→ 作者依据与反例
→ 可执行路径
→ 方法限制与适用边界
→ 下一步
```

长文不是把短帖注水。必要时给研究来源、术语定义和反例。

## 4. 标题与开头

先提取正文的真实承诺：读者身份、具体场景、核心区别、可得帮助和证据。生成 6 个候选后选 Top 3：

- **共鸣型**：让特定读者认出正在发生的时刻；
- **认知型**：指出一个容易混淆但正文能解释的区别；
- **搜索型**：使用读者会自然搜索的具体问题。

禁止：

- 用「抑郁症、NPD、人格障碍、创伤」等标签替代严谨判断；
- 「看一眼就知道」「三步治好」「根治」「保证有效」；
- 把正常情绪病理化，或通过吓人后果制造咨询需求；
- 标题承诺正文没有交付的诊断、疗效或科学证据。

开头优先使用场景、内心句、区别或作者的清晰判断，不用「在当今快节奏的社会」。

## 5. 服务承接

### CTA 梯度

按读者准备度选择一级：

1. **自我观察**：收藏一份问题、做一个小练习；
2. **继续了解**：阅读主页说明、公开文章、服务 FAQ；
3. **低压力互动**：留言一个非敏感关键词、领取不含诊断的清单；
4. **适配判断**：了解不同服务的区别、适合与不适合；
5. **预约行动**：按公开流程预约初谈、化学会谈或正式服务。

不要让读者在公开评论区披露创伤、诊断、关系身份或其他敏感经历。私信入口也要提醒不要先发送完整病史或会谈材料。

CTA 必须接住正文已经建立的信任程度。正文只完成“被认出”时，不突然要求预约高价服务；正文已讲清服务适配、过程和边界时，才进入预约行动。

### 服务表达七要素

1. 服务对象；
2. 当前问题与工作范围；
3. 服务形式和大致过程；
4. 从业者会提供的支持；
5. 来访者/客户需要投入的部分；
6. 适合、不适合、可能转介的情况；
7. 费用、取消、保密等信息的正式获取方式。

把「卖结果」改成「讲清潜在价值与工作过程」。可以说「帮助梳理、探索、练习、推进」，不要说「保证走出、彻底治愈、一定改变」。

发布服务稿前检查读者能否回答：为什么现在考虑、为什么可能找作者、服务中会发生什么、需要自己投入什么、什么情况下不适合、下一步如何了解。

## 6. 改稿与去 AI 味

按以下顺序检查：

1. 开头是否是一个人正在经历的时刻；
2. 是否一篇只讲一个判断；
3. 抽象词后是否有场景、动作、原话或依据；
4. 是否把假设写成诊断，把可能性写成必然；
5. 是否保留作者真实词汇、节奏、犹豫和边界；
6. 是否出现虚构案例、假反馈、假研究或万能结论；
7. CTA 是否与正文承诺一致且不利用脆弱。
8. 作者的独特判断是否被磨成任何同行都能说的套话；
9. 商业任务是否清楚，还是只在末尾临时加广告。

常见 AI 味：开头宏大空泛、连续「不是……而是……」、每段等长、机械三点法、抽象名词堆叠、假装共情、结尾突然升华。用户要诊断时，先指出具体句子与原因；用户明确要改稿时直接改。不要把“去 AI 味”理解成替换几个词，要恢复作者本人的判断和说话节奏。

## 7. 批量改编

先冻结「核心判断」，再分别改写：

| 平台 | 主要任务 | 证据密度 | CTA |
|---|---|---|---|
| 小红书 | 被发现、被识别 | 中 | 自查或继续了解 |
| 短视频 | 记住一个判断 | 低至中 | 一个问题或动作 |
| 朋友圈 | 形成持续信任 | 低 | 轻互动或服务说明 |
| 公众号 | 建立完整理解 | 高 | 阅读相关内容或了解服务 |

不要只改字数。每个平台重写开头、节奏、例子和行动入口。

## 8. 发布复盘

发布前记录：

- 希望吸引的具体读者；
- 预计最能让人停下的句子；
- 这条内容承担被发现、信任、服务教育还是预约；
- 它补的是内容到客户链路中的哪个断点；
- 最不确定的一个假设；
- 伦理或隐私风险是否已处理。

发布后观察：精准互动、主页访问、私信质量、服务问题、入口行动、适配咨询、读者误解和负面风险，不只看点赞。复盘时区分停留信号、精准信号、信任信号与商业信号，也区分内容质量、分发、时机和样本量；一次表现不升级为永久规律。
